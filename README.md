# Real-Time Shuttle Tracking Platform

## Overview

The Real-Time Shuttle Tracking Platform is a multi-tenant system designed to track shuttles in real-time. It supports four main actors: Admin, Organisation, Driver, and User, each with specific responsibilities and functionalities. The system is built using Golang and follows a clean architecture approach.

## Features

- **Admin**: Approves organization registrations, monitors active trips, and manages system controls.
- **Organisation**: Creates routes, assigns drivers, and manages stops dynamically.
- **Driver**: Starts trips using OTP, streams live location updates, and manages trip states.
- **User**: Tracks active shuttles based on organization and route codes.

## Architecture

The system is composed of several modules, including:

- **Domain Models**: Defines the core entities such as Org, Route, Stop, Trip, Driver, Location, OTP, and Audit.
- **Repositories**: Interfaces for database operations related to each domain model.
- **Services**: Business logic implementations for handling operations related to each domain.
- **Handlers**: HTTP handlers for managing API requests.
- **Middleware**: For authentication, logging, and error handling.
- **WebSocket**: For real-time location updates.
- **Event Publishing**: Using NATS for event-driven architecture.

## Setup Instructions

### Prerequisites

- Go (version 1.16 or higher)
- PostgreSQL
- Redis
- NATS
- OpenRouteService API key
- Google Geocoding API key

### Installation

1. Clone the repository:

   ```
   git clone https://github.com/akpatri/srt.git
   cd srt
   ```

2. Create a `.env` file based on the `.env.example` provided in the `config` directory and fill in the required values.

3. Run database migrations:

   ```
   go run ./migrations
   ```

4. Start the server:

   ```
   go run ./cmd/server/main.go
   ```

### API Documentation

API endpoints are documented using Swagger. You can access the documentation at `/docs/swagger.yaml` after starting the server.

### Usage

- **Admin**: Use the admin endpoints to manage organizations and monitor trips.
- **Organisation**: Register and manage routes and drivers through the organization endpoints.
- **Driver**: Start trips using the provided OTP and send location updates.
- **User**: Access active shuttles based on organization and route codes.

## Observability

The system includes observability features using OpenTelemetry for tracing and Zap for structured logging. Metrics can be collected to monitor system performance.

## Security

The platform implements JWT authentication, role-based access control (RBAC), and session-bound OTP for secure operations.
